# Mountain Boosted design language for dashboard UI

The dashboard adopts the visual language defined in `STYLEGUIDE.md`, derived from the Mountain Boosted base16 scheme and the same design principles used in the author's Emacs configuration. This means: no borders or box-shadows (separation by background color contrast only), a five-level visual weight system where most of the UI whispers, no rounded corners beyond minimal, no accent colors on static elements (accents signal state changes only), and tight internal spacing with generous outer padding. The categorical chart palette uses the scheme's warm accent colors (rose, gold, olive, green, steel, lavender, mauve).

This is a deliberate rejection of the common "rounded blue cards with drop shadows" pattern that most AI-generated dashboards default to. The dashboard should look like a data tool that belongs in the same computing environment as the terminal and editor, not a SaaS marketing page.
